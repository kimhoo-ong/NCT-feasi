NCT Feasibility Input Tool

A browser-based tool for entering and versioning property development feasibility data — GDV, cost breakdowns, and key returns — across multiple phases, sub-phases, and product components.

No installation Cloud save via Supabase Excel export Multi-phase · Multi-component

Quick Start

1

Fill Project Details

Click ▶ Project Details to expand the header. Enter company, plot, project name, and select a stage (BD → R1 → R2 → R3).

2

Enter Data in the Matrix

Add components (columns) and fill in units, NFA, ASP, and cost items. Numbers compute live as you type.

3

Save & Export

Click Save to store a versioned snapshot to the cloud. Click Report to download the Excel matrix.

Data Hierarchy (L1 – L5)

The tool organises data in a five-level hierarchy. Each level narrows the scope from company down to individual product type.

L1–L2 are entered once in the Project Details header (company name, plot code, location). L3 is a phase tab. L4 is a sub-phase card within a phase. L5 is a component column in the matrix — each product type (Residential, SOHO, Commercial, Car Park, etc.) is a separate column.

The Input Matrix

The main input area is a unified matrix. Each row is an attribute; each column is a component (L5). A TOTAL column on the right aggregates all components.

📑

Phases (L3) — Tabs

Click + next to the phase tabs to add a new phase. Each phase has its own KPI bar and sub-phase list. The ∑ Total tab shows a cross-phase rollup.

You can rename a phase by editing the tab — just click the tab label and type.
🗂

Sub-phases (L4) — Cards

Each phase can contain multiple sub-phases (e.g. Sub-phase A, Sub-phase B). Click + Add Sub-phase (L4) at the bottom of a phase. Each sub-phase card has:

  • A collapse / expand toggle (▾)
  • A Common Cost section (see below)
  • A component matrix (see below)
💰

Common Cost

Costs that belong to a sub-phase as a whole (not tied to one specific component), such as holding cost or project management fees. Click ▶ Common Cost above the matrix to expand.

Each Common Cost item has:

  • Pillar — which cost pillar this belongs to (P1–P5), so it rolls up correctly in the matrix
  • Code — item code, e.g. L.HOLD
  • Description — free text label
  • Amount (RM) — total sub-phase level cost

The amount is automatically allocated to each component column proportionally by NFA and included in that component's pillar total. No manual split needed.

Common Cost does not appear as a separate row in the matrix — it is folded into the correct pillar row so GDC totals are always consistent.
📊

💾

Save & Load

  • Save — writes the entire current state (all phases, sub-phases, components, costs) as a new versioned snapshot to the cloud. Each save is independent — nothing is overwritten.
  • Load — opens a list of all saved snapshots (project name · stage · date). Click any row to fully restore that snapshot into the tool.
To create a new revision: Load an existing snapshot → change the Stage → edit numbers → Save. The original snapshot is preserved.
📥

Export — Report

Downloads an Excel file ({ProjectCode}_{Stage}_Report.xlsx) with one sheet per phase, laid out as a matrix matching the on-screen layout:

  • Rows = attributes (Units, NFA, ASP, GDV, cost items …)
  • Columns = components + TOTAL
  • Cost items shown inline under each pillar with per-component breakdown
  • A Summary sheet rolls up Net GDV, GDC, PBT, and Margin across all phases
📤

Export — Pipeline (Fabric / Power BI)

Downloads a multi-sheet Excel file with flat normalised tables, ready for import into Microsoft Fabric or Power BI:

SheetContents
tbl_METAProject header — one row
tbl_PHASEPhase list with sort order
tbl_SALESOne row per component — all sales inputs and computed GDV figures
tbl_COSTOne row per cost line item per component, with pillar code and item code

Tips & Notes

💡

General Tips

  • All figures compute live as you type — no need to press Enter or click Recalculate.
  • The KPI bar at the top of each phase (Net GDV, GDC, PBT, Margin) updates instantly.
  • The breadcrumb at the top tracks L1 → L2 → L3 → L4 → L5 as you fill in names.
  • Pillar rows are expanded by default. Click the pillar header to collapse. Default line items are pre-filled — just enter the amounts.
  • Common Cost items are optional. If there are no sub-phase-level costs, leave the section collapsed.
  • The ∑ Total tab aggregates additive figures (GDV, GDC, PBT, units) across all phases. Non-additive ratios (margin, PSF) are recomputed from the totals.
  • Saving requires an internet connection (data is stored in Supabase cloud). The Report and Pipeline exports work fully offline.
